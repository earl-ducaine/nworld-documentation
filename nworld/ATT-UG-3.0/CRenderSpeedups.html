Increasing Render Speed

There are a number of features in N-Render, but it's somewhat difficult to say that any one is more expensive (in terms of render time) than another. This chapter describes some basic rendering techniques for getting the best performance out of N-Render.

Basics

  • When creating materials, be aware of your memory usage. Every map loaded into memory takes up some space, reducing the amount of memory left for N·World left to run, and potentially slowing down N-Render.

  • If you load a lot of texture in to memory at a high resolution you will use all your memory up and the machine will run slow.

  • Shaders have little affect on render time unless you are using raytracing, visibility, or face control. You can create solid color materials using shader parameters-there is no need for single color texture maps unless you are planning to use N-Paint 3D on it.

Mapping Questions

  • Texture maps have a minimal effect on render time, as long as the texture map resolution is not too high.

  • What is too high? Well, if your rendered image is 640x480, then you shouldn't need any texture map bigger then 512x512. (The only time you might need to use a larger map is if the object gets very close to the camera and you want the detail of the map to hold up).

  • Another mapping technique is to create seamless texture maps that repeat over and over with no apparent border. This lets you seamlessly texture an object without using additional memory. Creating seamless texture maps is described in the N-Paint Tutorial.

  • Once you have a seamless texture map, you can force the UVs of the mapper to repeat so that the texture is applied in a regular pattern over the surface of the object. For example, if you switch from a 512x512 texture to a 256x256 texture that has a UV scale of 2,2, you will use only 1/4 of the memory.

  • Bump maps can double or triple the render time of your scene-use them wisely.

  • Reflection and refraction maps can be used to simulate the raytracing effects possible in N-Render. Reflection maps do not slow down N-R render very much.

  • Raytracing can slow down your renders significantly. Use reflection or refraction maps if they will give you the same effect as raytracing-you'll save a lot of render time.

  • Opacity maps on objects that are close to the camera will make the render very slow (since the render must render "clear" pixels).

  • The more light you have in you scene will have an affect on the render time. Raytrace lights will also double or triple the render time.

Things to Remember

The more complex your scene, the longer the render. Adding to the number of faces in the scene always adds to the render time.

Rendering objects very close to the camera has a drastic affect on rendering time (when using complex materials).
