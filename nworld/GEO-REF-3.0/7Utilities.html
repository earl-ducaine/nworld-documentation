Utilities

The Utilities menu lets you do the following:

  • Restructure objects

  • Rename objects

  • Display information about objects

  • Render the current N-Geometry scene

  • Set operational preferences for N-Geometry

  • Customize your menu display

Figure 7.1 Utilities menu

Restructure

Collects several objects to form one new compound object. For example, if you have created a door by making three separate objects-the door itself, a handle, and a window-they can be collected and made into one named object with this command. The new object is referred to as a "group."

Such a restructured object can then be animated in N-Dynamics by referencing the new compound object rather than animating each terminal object separately. (Refer to the N-Dynamics Reference Manual and N-Dynamics Tutorial for more information.)

Using the same technique, you can attach a light to an object in order to illuminate a particular part of the object. If you later transform that object, the light will be also be transformed, remaining in its relative position to the object.

To restructure an object:

Rename

Renames an object. N-Geometry assigns an object a unique name, such as Cube when it is added to the environment. However, an assigned name does not necessarily reflect the nature of the object, and a name that describes the initial state of an object with relative accuracy such as cube or sphere might become inappropriate once the object has undergone extensive modification. Rename enables you to keep an object's name descriptive of its current state or its intended use.

  • (CLICK-L) changes the name of a single object.

  • (CLICK-M) on Rename lets you modify one or more objects at once, by specifying a prefix, suffix, or other random search and replace string. After choosing several objects the following menu is displayed:

Figure 7.6 Rename menu


    • Prefix for Name prepends the specified string to the selected objects. For example, a string of "big-" applied to a selected object "door" would change the object's name to "big-door".


    • Suffix for Name is identical to Prefix for Name, except that the specified string is appended to the selected objects.


    • Replace String lets you replace all occurrences of a string in the object list with another string. Enter the string you want to search for, press RETURN, then enter replacement string and press RETURN again.


    • Modify which lets you specify which types of objects you want to rename.

  • (CLICK-R) changes the name of several objects. After selecting several objects that you want to rename, a menu is displayed showing the old name for each, and displaying a text box for the new name of each.

Info

The Info operation lets you display various types of information about objects in the N-Geometry window. If you (CLICK-L) on Info, a menu of operations is displayed:

Figure 7.7 Info operations

Each of these operations is described below.

Set ?

Sets the global variable "?" to the last object, and the previous object to "??". This operation is typically reserved for use by developers.

For details on how to use this command, see the N·World Developer's Reference Guide.

Annotate

 The Annotate item prints element serial numbers over the line drawing of the object. The serial numbers will disappear by hitting the SPACEBAR, moving the camera, or moving the mouse off of the view window.

  • (CLICK-L) prints serial numbers for all visible vertices within the selected object.

  • (CLICK-M) prints serial numbers for all edges.

  • (CLICK-R) prints serial numbers for all faces.

Figure 7.8 Icosahedron with edges annotated

In addition to the above options, you may (CTRL-*) on Annotate display pertinent information about elements on the screen.

Using this operation requires some knowledge of the LISP format function and its directives, and should only be used if you need a special type of element display. For more information on this option, contact Nichimen Graphics Support.

  • Format String specifies the template for printing the information. It corresponds to the second (string) argument of format.

  • Format Args lists The values that are inserted into the above string according to the rules of the format function. The forms are evaluated inside each element as it is being annotated.

Show Data

Displays known information about an object or the camera.

  • (CLICK-L) prints geometric and topological information.

  • (CLICK-R) prints screen coordinate data for an object.

  • (CTRL-L) prints information to a file.

  • (CLICK-L) on Render to render the current scene.

  • (CLICK-R) displays the render defaults menu, where you can modify your defaults before rendering the scene. These parameters are described in more detail in the Attributes Editor User's Guide.

Preferences

The Preferences command opens a multi-page menu which lets you set global operational parameters for N-Geometry.

  • (CLICK-L) on Utilities>Preferences to display the Preferences menu. This menu defines a large number of operational parameters for N-Geometry.

Note. Some of these menu items are also found on the Modes submenu (see the section "Modes," on page 5-2), which typically comes up faster than this menu).

Across the bottom of the preferences menu are several buttons which let you save or change your preferences:


    • Set Preferences saves the preferences you've defined for the current session, but not to file.


    • Set & Save saves the preferences you've set to file. These become your defaults whenever you boot up N-Geometry.


    • Current undoes any changes you've made to the current menu without exiting the Preferences menu.


    • Defaults reverts to the user's saved preferences.


    • Abort exits from the Preferences menu without saving any changes.

To open a section of the menu, (CLICK-L) on the vertical bar with the appropriate name. Each submenu on the Preferences menu is described below:

Camera Preferences

  • Orthographic Mode switches between perspective and orthographic camera display modes.

Note. Depending on your renderer, you may or may not be able to render orthographic views. N-Render does support orthographic renders.
Note. You can also toggle orthographic mode on and off with (SHIFT-O)
Figure 7.10 illustrates two views of the same object, one in perspective and one in orthographic mode.

Figure 7.10 A perspective view and an orthographic view of the same object

  • Screen Stretch Factor controls the shape of pixels. The shape of monitor pixels varies, depending on the internal adjustments of each monitor. Most black-and-white console monitors do not have square pixels. Screen Stretch Factor compensates for the spatial asymmetry in x and y by stretching in x. The value should vary for each type of terminal and the particular settings of each monitor. The default is 1.0.

  • Auto Hither/Yon specifies whether hither and you clipping planes are calculated automatically.


    • If set to No, the default, the hither and yon clipping distances remain constant.


    • If set to Yes, the planes are continuously adjusted based on the position of objects in the previous frame.

    With auto-adjusting, at least part of an object must be visible or the hither and yon clipping planes are not updated. You should set Auto Hither/Yon to No when parameters such as eyepoint and location of object are explicitly set, since explicit settings might move the object beyond the hither and yon clipping planes.

    To set the hither and yon planes, (CLICK-R) on zoom +.

  • Range Expansion sets the hither and yon clipping planes to a certain distance in front of and behind the geometric points that are closest and farthest when Auto Hither/Yon is on. The amount of "headroom" is specified by this ratio, which can be a number from 0 to 1. In Figure 7.11, this is the ratio of distance a to b:

Figure 7.11 Setting the viewing depth ratio

Note. Keeping the amount of headroom to a minimum generally increases the quality of rendered images.

  • Min Hither Distance defines the minimum hither distance for in effect. Even with Auto Hither/Yon set to Yes, N-Geometry sets a minimum hither distance from the eye.

Note. A very low minimum hither distance reduces the depth precision with which objects are rendered; objects may appear to overlap or be displayed out of order as they get extremely close to the viewer.

Figure 7.12 Hither and yon clipping planes

  • World Units specifies the unit by which distances and sizes are measured. The default is FEET. You may also use microns, millimeters, centimeters, meters, kilometers, inches, yards, and miles.

    • NEW lets you create your own unit, expressed in terms of metric conversion.

    Changing this value scales all the objects in the world and the physical location of the camera. The effect is not visible, but the coordinates of all bodies are changed.

  • Camera Format defines the characteristics of your viewport, and is particularly useful when creating rendered scenes you wish to composite, for example, with live footage. If you know the camera settings used to generate the live footage, you can select that format and be sure that the two will match.

Mouse Preferences

  • Mouse Speed controls how fast the cursor moves in relation to movement of the mouse. Decreasing this value makes your mouse move slower; raising the value makes it more responsive.

Note. It may be helpful to lower the mouse speed if you are doing detail work that requires fine movement.
The default is 25.

  • Mouse Accelerator accelerates camera motion and the like as you move the mouse faster. 0 is linear (the speed of the motion is proportional to the speed of the mouse), and increasing the value increases the relative speed of the motion. (This parameter works together with Mouse Speed above).

  • Translation Factor applies a factor (multiplier) to the mouse speed for translation operations, to control how much a selected element is moved. Controlling the speed of a move operation is helpful in situations when you have zoomed in on an element. If the factor is set to 0, the mouse won't move. The default is 1, normal mouse movement.

  • Shift Key Factor accelerates mouse movement by the specified factor If you hold the SHIFT key down.

  • Monitor Operations enables the mouse monitor window.

  • Track Serial Numbers? if selected, displays the serial number for the currently selected element in the upper left corner of the N-Geometry window.

Pathname Preferences

  • Global Default is the default directory that N-World products looks to first for any objects.

Note. The global default directory is common to all N-World products.

  • Font Data is the directory where Bitstream font information is stored.

  • Default Object is the directory displayed by default when loading or saving objects. The system default is the user's home directory.

  • Default View is the directory displayed by default when loading or saving views.

  • Relative Bodyfile is used if you save objects using separate body files the File>Save menu. The default is bodies/, which causes all body files to be saved in a subdirectory right beneath the directory where objects are saved.


    • Auto Create? specifies whether relative bodyfiles are created automatically. If the specified directory cannot be found with Auto Create? set to Yes, the file is created automatically as well. If this parameter is set to No, you are prompted to specify the directory in which the relative body files are saved.

Element Edit Preferences

  • Scale by Zero Action lets you define what happens when you try to scale an object with a zero:


    • Ignore allows scaling of objects by zero without warning, regardless of potential problems.


    • Notify allows scaling of objects by zero, but displays a warning in the UNIX shell from which N-Geometry was started.


    • Ask displays a pop-up dialog asking you to either confirm or enter a new value for the scaling operation. This is the default.


    • Replace automatically replaces zero with the value found in the LISP form 3D::*DEFAULT-SCALE-BY-ZERO-STANDIN*.

  • Scale by Zero Standin lets you specify a default value used with the Replace option in the Scale by Zero Action (directly above).

  • Magnet Falloff Function specifies the curve type used to calculate area of influence calculations.


    • Straight uses an inverse linear curve; influence decreases linearly from the center.


    • Spike uses an inverse square curve; influence decreases quickly near the center, more slowly at the edge.


    • Bell use a cosine curve; influence decreases slowly at the center and edge, quicker in between.


    • Dome uses a half-cosine curve; influence decreases gradually at the center, more quickly at the edge.

  • Magnet Distance Route determines how areas of influence are measured:


    • Shortest uses the shortest direct distance from the periphery of the selected element.


    • Midpoint uses the shortest direct distance from the midpoint of the selected element.


    • Surface approximates the shortest distance following face surfaces.

  • Numeric Input Default specifies whether numeric input menus are displayed by default for modify operations.

  • Tube Transforming Item determines whether an additional field appears on the tube creation dialog box. (Advanced users can then specify a function in that dialog box that modifies the cross-section of the tube as it moves along the path. For more information on using this feature, contact Graphics Support.)

  • Logo Utilities Item, if selected, adds items to the text menu for automatic extruding, deholing, beveling, and coplanaring of letters, all useful functions for creating specialized type like that used in logos.

  • Max Face Size for Update is the maximum number of vertices a face can have and still have its normals updated interactively.

  • Max Normals for Update specifies the maximum number of normals that are updated during interactive operations (e.g., Move). It primarily affects backfacing. If the operation would cause more than the specified number of normals to be updated, backfacing display is not updated until after the operation is completed.

  • Min Edge Length is the smallest length an edge can have. If N-Geometry is told either directly or indirectly to cut an edge that would produce an edge smaller than this number, the operation is not performed.

  • Max Face Bevel Ratio is used for Bevel and Coplanar operations. If the newly created faces would cut into the original face by more than this amount (expressed as a fraction of the face size), they are trimmed back. The default is one-third.

  • Coincidence Tolerance is used for wires. For certain operations, the closest two points can be before they are considered to be in the same place.


    • for Weld specifies the maximum distance elements may be separated by for the Weld operation to "take."

  • Default Arc Subdiv Angle is used when converting arcs to wires. This is the interval at which the wires are to be subdivided.

  • Min Bounds Ratio is used with the Scale to Fit operation. It specifies the minimum ration maintained between the two objects when performing the operation (which prevents objects from being scaled to values close to zero). This is of particular value when scaling objects that are vastly different in size.

  • Allow Doubled Edge Cuts specifies whether N-Geometry allows you to cut a face along an existing edge, creating a double edge. Double edges are visually indistinguishable from single edges, but are typically undesirable, especially when rendering.

  • Copy Bridgee? specifies whether Bridge operation ((CTRL-*) to another body) copies the second body and leaves the original untouched, or absorbs it into the body bridged from.

Object Preferences

  • Shutdown Save Objects determines what N-Geometry does with objects when you shut down:


    • All automatically saves all objects to their default names.


    • New saves all new or modified objects.


    • Ask queries you about each object before shutting down.


    • Menu displays a menu and prompts you to specify which objects to save. This is the default.


    • None shuts down without saving any objects.

  • Object Addition Mode determines whether the latest object is added to the bottom (the default) or top of object lists.


    • Append adds the object to the bottom of the list.


    • Prepend adds the object to the top of the list.

  • Default Rewrite Bodyfiles? specifies whether Force Write New? is set to Yes or No by default on the File>Save menu when writing objects out with separate body files.

  • Default Primitives Size sets the default size (in world units) of ready made polyhedra. The default is 10.

  • Collectability Test determines which elements are collected when using the lasso to make a collection.


    • Drawn selects all lassoed points regardless of their sensitivity status.


    • Drawn + Sensitive tests for sensitivity of points before including them in a collection (points with sensitivity off are not included in a collection).

  • Sensitivity Threshold is the maximum number of points a newly added object can have and still start out "sensitive." (Objects added to N-Geometry that have more than this target number of vertices are loaded in "insensitive," speeding up N-Geometry.)

  • Restructure Renames Inferior? specifies how to name objects created by making a multiple object out of a terminal object.


    • If Yes, restructuring A into terminal object B causes A and
      Essential-B to be grouped under B.


    • If No, A and B are grouped under B Group.

  • Copy Copies Appearance specifies whether a copied object should use the same display parameters (e.g., shading and visibility) as the object it was copied from or use the default parameters for the object type as found in the Utilities>Preferences>Default Display menu.

  • Sticky Standin Bodies? substitutes boxes for selected objects at all times. This can significantly speedup redisplay in N-Geometry if you are working with a large object.

  • Point Display Radius specifies the radius, in pixels, of a point object (such as a scaling center). The valid values are 1 to 4. The default is 3, and higher values act as if you specified 4.

  • Segment Highlight Width lets you specify the width of highlighted segments when working in Segments mode. The default is 3.

  • Light Strength Radius defines the size of the "sunburst" around a light in the N-Geometry view window.

Note. This parameter does not effect the brightness of the light, only its appearance in the interface.

  • Highlight Segment Heads specifies whether or not to display a dot at the head end of a segment, edge, or instance of a related flavor as it is highlighted so you can see its orientation. While the orientation of a segment is usually not important, it can be significant when you are selecting a segment to act as an axis.

  • Draw control points solid? specifies whether the wires that control the
    shape of a Bezier or spline are drawn as wires or as points.

  • Highlight Hidden determines whether or not hidden elements are displayed (e.g., can you see through a wire-frame cube or is it opaque?).


    • Yes highlights points and segments that are collected but currently hidden from view.


    • No, the default, highlights only those collected points and segments that are currently visible.

  • Visible Attachments determines whether object attachments (such as scaling centers, rotation axes, and other sub-objects) are Visible or Hidden. (Display of attachments can also be set from the Modes submenu.) The default is Visible.

Note. This parameter overrides an elements Visibility parameter (set using Object Display>Visibility).

Debug Preferences

These options are for users of the Nichimen Developer's Kit only. Refer to the documentation for that product for more information on these options.

Global Preferences

  • Global GC Behavior determines when and how a gc (garbage collection) operation is performed on the machine running N-Geometry.


    • Notify displays a message that garbage collection should be performed.


    • Auto performs a garbage collection whenever needed, without prompting the user that the operation is needed.


    • Auto + Warn displays a warning and automatically performs the required garbage collection.


    • No GC specifies that no garbage collection is to take place.

  • GC Threshold (bytes) specifies the number of bytes that are tenured before the specified GC behavior is executed.

  • Print Progress specifies how output sent to the UNIX shell (e.g., error descriptions) should appear.


    • None sends no output to the shell.


    • Notes sends descriptions only.


    • Time + Notes sends descriptions and a timestamp for each message.

  • Error Menu determines what options are available if an error occurs:


    • Simple displays a basic error message menu.


    • Expert gives the additional option of entering the LISP debugger (useful for developers).


    • None goes directly to the LISP debugger.

  • Describe Errors, if selected, provides a more verbose description of the error messages generated.

Custom Menus

Lets you control what operations appear on N-Geometry's modify menu; there are three pre-defined modes for menus.

When you (CLICK-L) on GeoMenus>Utilities>Custom Menus you can select from one of the following pre-defined menu modes:

Long

Displays all possible commands for the selected element type on the modify body menu; more commonly used commands appear in the left column, less common on the right (for optimized flip menus).

Short

Displays the most commonly used commands on a single page menu. (This mode is useful if you are just getting familiar with the system.)

By Category

Displays menus as they appeared in N·World 2.1 and earlier software releases; experienced users may want to use this mode if they are familiar with the location of certain commands in the menu.

Edit Menus

The Edit Menus option lets you define your own custom menu configurations; you select which commands are to be included and name that configuration.

This is handy whether you have a particular set of commands that you commonly use, or if you are working on a specialized task and want to create a menu with only a few operations on it.

To create a custom menu:

1. (CLICK-L) on GeoMenus>Utilities>Custom Menu.

2. (CLICK-L) on Edit Menus.
