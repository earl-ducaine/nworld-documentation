About this Document

The Nichimen Open World Programmer's guide is designed to provide a broad understanding of the concepts necessry to develop Lisp programs for N·World.

Contents of This Document

  • Chapter 1, "Welcome to Nichimen's Wide Open World" is the place to go for a quick start with Wide Open World. It provides a brief introduction to WOW, it's major components, and to Lisp programming.

  • Chapter 2, "Lisp Fundamentals" provides a brief overview of important Lisp concepts, including the Lisp interpreter, symbols, forms and syntax, lists, and functions

  • Chapter 3, "Creating N·World Code" describes how to launch Xemacs, run N·World from Xemacs, and how to create, compile, and store your own code as plug-ins. It also includes an introduction to N·World packages.

  • Chapter 4, "Defining Foreign Functions" summarizes the procedure for defining foreign functions in N·World.

  • Chapter 5, "Building Interfaces with the UI-Toolkit" describes how to use the N·World UI Toolkit to build menus, pop-up dialog boxes, and other tools for interacting with users.

  • Chapter 6, "Introduction to N·World Objects" describes the basic structure of an N-Geometry object, the N-Geometry object hierarchy, and N-Geometry coordinate systems.

  • Chapter 7, "Creating, Manipulating, and Traversing Bodies and Objects" includes important concepts for creating, selecting, viewing, and manipulating bodies, objects, and groups of elements.

  • Chapter 8, "Exporting Object Data" describes how to export basic geometrical data, UV coordinate data, and transformation information from bodies, elements of bodies, objects, and object hierarchy. It also describes some basic techniques for directing and formatting this data.

  • Chapter 9, "Rendering Operations" describes basic techniques for defining new attribute domains, attribute sets, and attributes. This chapter also covers adding your new domain to the Attributes Editor

  • Chapter 10, "Extending Applications"describes how to extend N-Geometry by defining new edmethods. It also shows how to extend N-Dynamics by defining your own dynamics operations. Dynamics operations are performed on objects in each frame of a Dynamics animation.

  • Chapter 11, "Traversing Elements"describes the predefined iteration paths used to traverse elements of N-Geometry's database.

  • Chapter 12, "About this Document" describes how you can increase the performance of your code with anti-generic functions. It also describes how to create and manipulate ftriplets.

  • Appendix A, "ASCII Object File Format" describes the file format for objects saved in ASCII format.

  • Appendix B, "N·World 2.1 - 3.0 Renamed Functions" is a concordance showing the new names for functions and symbols which were renamed during the upgrade to N·World 3.0.

Document Style Conventions

This document contains a number of style conventions designed to make it easier to use. Different typefaces indicate such things as screen messages and operator entries. The following sections describe the style conventions used.

Functions

Functions are presented in capital letters when first presented. Packages are always included with double-colon syntax. This syntax is not always required, but including it ensures that functions will perform as written regardless of the current Lisp package.

Arguments

Optional arguments, which are not required for the function to work, are in italic text. Punctuation outside of the braces must be included, while everything inside the braces is purely descriptive. In this example:

(PackageA::FooFunc arg-1 :arg-2 `arg-3 (list))
  • Arg-1 is required

  • Arg-2 is also required, and must be preceded by a colon (it is a keyword).

  • Arg-3 is optional. If provided, it must be quoted.

  • List symbolizes an argument that is a list.

Notes

Notes of special interest are set off from the text within horizontal rules:

Note. Notes provide information that is important enough to demand special attention. The information may be relevant to the current discussion but awkward to include without disrupting the flow of information.

Mouse and Keyboard Notation

The following conventions are used in the text:

Table 1 Style conventions
Style Description
(shift-L)

(CTRL-L) (ALT-L)

(Shift-ctrl-L)

 Key combinations are enclosed in parentheses. (shift-L) means to click the left mouse button while holding down the shift key.

Special modifier keys are also enclosed in parentheses.

Combinations of modifier keys are also possible.

Menu Notation

In N-Geometry, menu items that are preceded and followed by a period have multiple selections, which you select by clicking left, middle, or right on the mouse. (The different options are shown on the mouse documentation line, displayed at the bottom of the screen.)
